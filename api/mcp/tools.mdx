## Available Tools

The PairOfCleats MCP server exposes the following tools:

- [index_status](#index_status) - Check index and cache status
- [config_status](#config_status) - Inspect configuration and warnings
- [build_index](#build_index) - Build or update indexes
- [build_sqlite_index](#build_sqlite_index) - Build SQLite indexes
- [compact_sqlite_index](#compact_sqlite_index) - Compact SQLite databases
- [search](#search) - Run search queries
- [search_workspace](#search_workspace) - Federated workspace search
- [triage_ingest](#triage_ingest) - Ingest vulnerability findings
- [triage_decision](#triage_decision) - Create triage decisions
- [triage_context_pack](#triage_context_pack) - Generate triage context
- [download_models](#download_models) - Download embedding models
- [download_dictionaries](#download_dictionaries) - Download dictionary wordlists
- [download_extensions](#download_extensions) - Download SQLite extensions
- [verify_extensions](#verify_extensions) - Verify extension availability
- [cache_gc](#cache_gc) - Garbage collect caches
- [clean_artifacts](#clean_artifacts) - Remove cache artifacts
- [bootstrap](#bootstrap) - Bootstrap complete setup
- [report_artifacts](#report_artifacts) - Report artifact sizes

## index_status

Return cache and index status for a repository.

### Parameters

<ParamField path="repoPath" type="string">
  Repository path (defaults to server cwd)
</ParamField>

### Response

Returns a detailed status report including:

- Repository metadata (path, ID, git status)
- Cache directories and paths
- Index artifact status (code, prose, records)
- SQLite database status
- Model and dictionary availability
- Metrics file locations

### Example

```json
{
  "repoPath": "/path/to/repo",
  "repoId": "abc123",
  "git": {
    "isRepo": true,
    "head": "a1b2c3d4",
    "branch": "main",
    "isDirty": false
  },
  "index": {
    "code": {
      "dir": "/cache/repo/index-code",
      "chunkMeta": { "exists": true, "mtime": "2024-03-04T12:00:00Z", "bytes": 1048576 },
      "tokenPostings": { "exists": true, "mtime": "2024-03-04T12:00:00Z", "bytes": 524288 }
    }
  }
}
```

## config_status

Inspect configuration and cache status with capability warnings.

### Parameters

<ParamField path="repoPath" type="string">
  Repository path (defaults to server cwd)
</ParamField>

### Response

Returns:

- Current configuration (all config blocks)
- System capabilities
- Cache status and availability
- Warnings for missing dependencies or configuration issues

### Example

```json
{
  "repoPath": "/path/to/repo",
  "capabilities": {
    "mcp": { "sdk": true },
    "watcher": { "parcel": false }
  },
  "warnings": [
    {
      "code": "model_missing",
      "message": "Embedding model not found. Run node tools/download/models.js."
    }
  ]
}
```

## build_index

Build or update indexes for a repository with optional SQLite and incremental support.

### Parameters

<ParamField path="repoPath" type="string">
  Repository path (defaults to server cwd)
</ParamField>

<ParamField path="mode" type="string" default="all">
  Index mode: `all`, `code`, `prose`, `extracted-prose`, or `records`
</ParamField>

<ParamField path="incremental" type="boolean" default={false}>
  Reuse per-file incremental cache
</ParamField>

<ParamField path="sqlite" type="boolean">
  Build SQLite indexes after JSON indexes (defaults to config setting)
</ParamField>

<ParamField path="stubEmbeddings" type="boolean" default={false}>
  Skip model downloads and use stub embeddings for testing
</ParamField>

<ParamField path="useArtifacts" type="boolean" default={false}>
  Restore CI artifacts before building
</ParamField>

<ParamField path="artifactsDir" type="string">
  Path to CI artifacts directory (required if useArtifacts is true)
</ParamField>

### Response

```json
{
  "repoPath": "/path/to/repo",
  "mode": "all",
  "sqlite": true,
  "incremental": false,
  "restoredArtifacts": false
}
```

<Note>
This is a long-running tool that emits progress notifications. Default timeout is extended for index builds.
</Note>

## search

  Search query string
</ParamField>

<ParamField path="repoPath" type="string">

</ParamField>

<ParamField path="output" type="string" default="compact">

<ParamField path="backend" type="string">

## search_workspace

Run federated search across repositories from a workspace configuration.

### Parameters

<ParamField path="workspacePath" type="string" required>
  Workspace configuration file path (.jsonc)
</ParamField>

<ParamField path="query" type="string" required>
  Search query string
</ParamField>

<ParamField path="workspaceId" type="string">
  Expected workspace repoSetId for cross-check validation
</ParamField>

<ParamField path="search" type="object">
  Single-repo search parameters forwarded to each repository (mode, top, backend, filters, etc.)
</ParamField>

<ParamField path="select.repos" type="string | string[]">
  Filter to specific repositories
</ParamField>

<ParamField path="select.tags" type="string | string[]">
  Filter by repository tags
</ParamField>

<ParamField path="select.repoFilter" type="string | string[]">
  Regex filter for repository selection
</ParamField>

<ParamField path="select.includeDisabled" type="boolean" default={false}>
  Include disabled repositories
</ParamField>

<ParamField path="limits.perRepoTop" type="number">
  Maximum results per repository
</ParamField>

<ParamField path="limits.concurrency" type="number">
  Maximum concurrent repository searches
</ParamField>

<ParamField path="merge.strategy" type="string" default="rrf">
  Result merging strategy (currently only `rrf` supported)
</ParamField>

<ParamField path="merge.rrfK" type="number">
  RRF (Reciprocal Rank Fusion) K parameter
</ParamField>

<ParamField path="cohort" type="string | string[]">
  Cohort filter for result mixing
</ParamField>

<ParamField path="cohorts.policy" type="string">
  Cohort policy: `default` or `strict`
</ParamField>
