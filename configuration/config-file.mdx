PairOfCleats uses a `.pairofcleats.json` file in your repository root to configure build, indexing, retrieval, and runtime behavior.

## File Location

Place `.pairofcleats.json` in the root of your repository:

```text
my-repo/
├── .pairofcleats.json
├── src/
└── ...
```

## Schema

The configuration file follows JSON Schema Draft 2020-12. The full schema is available at `docs/config/schema.json` in the source repository.

## Example Configuration

Minimal example:

```json
{}
```

PairOfCleats works with an empty configuration file using intelligent defaults.

## Top-Level Namespaces

<ParamField path="cache" type="object">
  Cache root and storage configuration
  
  <ParamField path="cache.root" type="string">
    Absolute path to cache directory. If not specified, uses platform-specific defaults:
    - Windows: `%LOCALAPPDATA%\PairOfCleats\cache`
    - macOS/Linux: `~/.cache/pairofcleats`
    
    Example:
    ```json
    {
      "cache": {
        "root": "C:/absolute/path/to/cache"
      }
    }
    ```
  </ParamField>
</ParamField>

<ParamField path="quality" type="string" default="auto">
  Build quality preset. Controls trade-offs between speed and completeness.
  
  Options:
  - `auto` - Automatic selection based on repository size
  - `fast` - Minimal indexing for quick builds
  - `balanced` - Recommended for most repositories
  - `max` - Complete indexing with all features enabled
</ParamField>

<ParamField path="threads" type="number">
  Number of worker threads for parallel operations. Defaults to available CPU cores.
</ParamField>

## MCP Configuration

<ParamField path="mcp" type="object">
  Model Context Protocol server settings
  
  <ParamField path="mcp.mode" type="string" default="auto">
    MCP server mode. Requires @modelcontextprotocol/sdk capability for `sdk` mode.
    
    Options:
    - `legacy` - Legacy MCP protocol
    - `sdk` - Modern SDK-based protocol
    - `auto` - Automatic detection
  </ParamField>
  
  <ParamField path="mcp.queueMax" type="number">
    Maximum queued MCP requests
  </ParamField>
  
  <ParamField path="mcp.maxBufferBytes" type="number">
    Maximum buffer size for MCP messages in bytes
  </ParamField>
  
  <ParamField path="mcp.toolTimeoutMs" type="number">
    Default timeout for MCP tool calls in milliseconds
  </ParamField>
  
  <ParamField path="mcp.toolTimeouts" type="object">
    Per-tool timeout overrides (tool name -> timeout in milliseconds)
  </ParamField>
</ParamField>
