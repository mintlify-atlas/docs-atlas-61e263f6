# Introduction to PairOfCleats

PairOfCleats is a hybrid semantic indexing and search utility built for source repositories. It combines deterministic artifact production with mixed sparse + dense retrieval to provide powerful code and documentation search capabilities.

## What is PairOfCleats?

PairOfCleats builds deterministic index artifacts for code and prose, then runs mixed sparse + dense retrieval with strict contracts around artifacts, schemas, and cache identity. It's designed as a local-first tool that gives you complete control over your search infrastructure.

### Two-plane architecture

PairOfCleats operates on two distinct planes:

- **Build plane**: Deterministic artifact production with contract-first artifacts and manifest-first loading
- **Retrieval plane**: Query planning, candidate generation, scoring, and output shaping with hybrid ranking

## How it works

### Build pipeline

```text
Repo files
  -> discovery + mode classification
  -> chunking + metadata + postings + relations
  -> artifact pieces + manifest + build_state
  -> optional sqlite/ann materialization
  -> builds/current.json promotion
```

The build process starts with file discovery and mode classification, then performs chunking and metadata extraction. Sparse and dense artifacts are written with manifest entries, followed by optional database materialization.

### Retrieval pipeline

```text
Query
  -> parse + plan + intent
  -> candidate prefilter
  -> sparse rank (BM25 / sqlite-fts)
  -> dense rank (ann providers)
  -> fusion + boosts + explain
  -> stable output (human or json)
```

Queries are parsed and planned, then candidates are generated through prefiltering. Both sparse and dense ranking are performed, with results fused using RRF or blend policies for optimal relevance.

## Index modes

PairOfCleats supports four specialized index modes:

- **code**: Source code files with syntax-aware chunking and tree-sitter parsing
- **prose**: Documentation, markdown, and plain text with semantic chunking
- **extracted-prose**: Content extracted from PDFs, DOCX, and other document formats
- **records**: Structured data from external symbol systems (ctags, LSIF, SCIP)
