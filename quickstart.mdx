# Quick start

This guide will walk you through installing PairOfCleats, setting it up for your repository, building your first index, and performing searches.

## Prerequisites

Before you begin, ensure you have:

- Node.js `>=24.13.0` installed (check with `node --version`)
- npm package manager
- A source repository to index

<Note>
PairOfCleats uses a specific Node.js version for optimal compatibility. The repository includes an `.nvmrc` file set to `24.13.0`.
</Note>

<Steps>
  <Step title="Install dependencies">
    Clone or download PairOfCleats, then install dependencies:
    
    ```bash
    npm install
    ```
    
    This command installs all required packages and applies necessary patches for native modules.
    
    <Tip>
    For production-only installs, note that dev dependencies are required for patches to apply correctly. Avoid using `npm ci --omit=dev` in this repository.
    </Tip>
  </Step>
  
  <Step title="Run guided setup">
    PairOfCleats provides an interactive setup wizard to configure models, tooling, and cache settings:
    
    ```bash
    pairofcleats setup
    ```
    
    The setup wizard will:
    - Configure embedding models and download required assets
    - Set up optional tooling (Python, tree-sitter grammars)
    - Configure cache root location
    - Verify SQLite extensions if needed
    
    **Alternative: Non-interactive bootstrap**
    
    For automated or CI environments, use the bootstrap command:
    
    ```bash
    pairofcleats bootstrap
    ```
    
    This performs a fast, non-interactive setup with sensible defaults.
  </Step>
  
  <Step title="Build your first index">
    Build indexes for your repository with the `index build` command:
    
    ```bash
    pairofcleats index build --mode all --quality balanced
    ```
    
    <CodeGroup>
    ```bash Build all modes
    pairofcleats index build --mode all --quality balanced
    ```
    
    ```bash Build code only
    pairofcleats index build --mode code --quality balanced
    ```
    
    ```bash Build prose only
    pairofcleats index build --mode prose --quality balanced
    ```
    
    ```bash High quality with embeddings
    pairofcleats index build --mode all --quality high
    ```
    </CodeGroup>
    
    The build process will:
    1. Discover and classify files by mode
    2. Chunk content and extract metadata
    3. Generate sparse artifacts (postings, chargrams)
    4. Optionally compute embeddings for dense retrieval
    5. Write manifest and promote to `builds/current.json`
    
    **Quality levels**:
    - `fast`: Sparse-only indexing, no embeddings
    - `balanced`: Selective embeddings based on heuristics
    - `high`: Full embeddings for all chunks
    
    <Note>
    The first build may take several minutes depending on repository size and quality settings. Subsequent incremental builds are much faster.
    </Note>
    
    **Validate the index**:
    
    ```bash
    pairofcleats index validate
    ```
    
    This verifies artifact integrity and manifest consistency.
  </Step>
  
  <Step title="Search your codebase">
    Now you can search your indexed repository:
    
    ```bash
    pairofcleats search -- "where is query cache invalidated?" --mode code
    ```
    
    <CodeGroup>
    ```bash Code search
    pairofcleats search -- "where is query cache invalidated?" --mode code
    ```
    
    ```bash Documentation search
    pairofcleats search -- "release matrix and packaging" --mode prose
    ```
    
    ```bash Search with explanations
    pairofcleats search -- "configuration schema" --mode all --explain
    ```
    
    ```bash JSON output
    pairofcleats search -- "error handling" --mode code --json
    ```
    </CodeGroup>
    
    **Query syntax**:
    - `"exact phrase"` - Match exact phrases
    - `-term` - Exclude a term
    - `-"excluded phrase"` - Exclude a phrase
    
    **Useful flags**:
    - `--mode <mode>` - Search specific modes: `code`, `prose`, `extracted-prose`, `records`, `all`
    - `--explain` - Show ranking details and pipeline stats
    - `--stats` - Display timing and memory checkpoints
    - `--json` - Output results in machine-readable JSON format
    
    <Tip>
    Use `--explain` to understand why results were ranked the way they were. This shows sparse scores, dense scores, and fusion weights.
    </Tip>
  </Step>
</Steps>

## Common workflows

### Watch mode for development

Automatically rebuild indexes when files change:

```bash
pairofcleats index watch
```

This is useful during active development to keep indexes up-to-date.

### Start the API server

Run PairOfCleats as an HTTP service for programmatic access:

```bash
pairofcleats service api
```

The API server provides endpoints for search, indexing status, and management operations.

### Custom cache location

Set a custom cache root in `.pairofcleats.json`:

```json .pairofcleats.json
{
  "cache": {
    "root": "/absolute/path/to/cache"
  }
}
```

This is useful for centralized cache management across multiple repositories.
